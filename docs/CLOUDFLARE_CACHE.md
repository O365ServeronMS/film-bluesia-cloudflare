# Cloudflare cache bindings

The FilmBluesia Worker keeps only the runtime bindings it actively uses.

## Required bindings

- `ASSETS`: serves Astro's generated static assets.
- `KV`: stores normalized OPhim metadata and refresh write-budget counters.
- `WORKER_VERSION`: Cloudflare Worker Version Metadata binding; its deployment `id` namespaces HTML Cache API keys automatically.
- `IMAGE_CACHE_BASE_URL`: public base URL for the shared external image cache (`https://img.bluesia.net`).
- `IMAGE_CACHE_SIGNING_SECRET`: server-only HMAC secret used to sign external image-cache URLs.
- `ADMIN_REFRESH_TOKEN`: server-only secret for the optional protected manual refresh endpoint.

The site-local R2 image binding was removed after active image delivery moved to the shared external cache. `IMAGE_CACHE_SIGNING_SECRET` must never use a `PUBLIC_` prefix.

## Automatic deploy cache isolation

`src/middleware.ts` includes `WORKER_VERSION.id` in every internal HTML Cache API key. Cloudflare creates a new version ID for every Worker deployment, so a newly deployed build cannot read HTML generated by an older deployment.

`HTML_CACHE_VERSION` remains a local/manual fallback only when Version Metadata is unavailable. It is not a production variable and no longer needs to be bumped for each commit.

Old versioned Cache API entries are left to expire under their existing TTLs; they are unreachable from the new deployment immediately.

## TTL policy

- Home, list HTML, API responses, and list/taxonomy metadata: `1800` seconds.
- Movie metadata: `7776000` seconds.
- Movie HTML: `7776000` seconds for completed/full titles with playable links; `86400` seconds otherwise.
- Search: no-store.
- External images: owned by `img.bluesia.net`, not this Worker.

Favorites, watch history, and settings remain browser-local and are excluded from cached HTML.

## Scheduled OPhim refresh

`wrangler.jsonc` runs the refresh every two hours:

```json
"triggers": {
  "crons": ["0 */2 * * *"]
}
```

The Worker defaults to 24 movies per run and a 1500 ms delay. These defaults live in code, so duplicate production variables are unnecessary. Refresh writes use stable-hash deduplication and daily KV write budgets.

## Manual OPhim refresh

`POST /api/admin/refresh` requires `x-refresh-token: <ADMIN_REFRESH_TOKEN>`, returns `401` without a valid token, and is rate-limited through KV.

```powershell
curl -X POST https://film.bluesia.net/api/admin/refresh `
  -H "content-type: application/json" `
  -H "x-refresh-token: $ADMIN_REFRESH_TOKEN" `
  -d '{"mode":"latest","force":false}'
```

## Production binding inventory

After deployment, the intended runtime inventory is:

- bindings: `ASSETS`, `KV`, `WORKER_VERSION`
- plain-text variable: `IMAGE_CACHE_BASE_URL`
- secrets: `IMAGE_CACHE_SIGNING_SECRET`, `ADMIN_REFRESH_TOKEN`

Do not add site-local image proxy/R2 variables back unless the architecture explicitly moves image delivery away from the shared cache.
